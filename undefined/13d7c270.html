#Course Description

This course is designed to provide a solid understanding of the fundamentals of database technology. The course lays special emphasis on database development process, data modelling using Entity relationship diagramming, Database normalization, data definition and manipulation using Structured Query Language (SQL), issues of performance tuning and security. The course adopts a hands on approach and provides students with experience of tools such as Oracle data Modeler, command line interpreters and GUI utilities for SQL along with Query analyzers and profilers. Students will also be exposed to front end programs and database integration using platforms such as Java, Dot Net and PHP. The course also provides a brief overview of advanced technologies such as data warehousing. The course also develops student’s ability to work in team by engaging in team and group projects.

Learning Outcomes

\1. Students will develop a comprehensive understanding of the database development process including analysis, design, implementation and testing

\2. Students will demonstrate proficiency in enterprise data modelling using Entity relationship diagrams, translating conceptual models into relational models. Students will create models using Oracle data modeler

\3. Students will understand database normalization including definition, process of normalization and the various anomalies that normalization overcomes. They will apply the techniques to come with a normalized data model

\4. Students will demonstrate strong proficiency in SQL including advanced SQL constructs such as joins, sub queries and recursive queries using command line and GUI utilities

\5. Students will develop a good understanding of indexes and procedures and how they can be used to tune the performance of databases. They will be exposed to tools such as profilers and Query analyzers

\6. Students will understand what data warehouses and data marts are and the difference between OLTP(Online transaction processing systems) and OLAP(Online analytical processing systems)

Course Requirements:

Labs: will help develop hands on experience. Students must complete these to prepare for assignments and projects. Labs will not be graded.

Assignments: Four assignments will help reinforce concepts from lectures, labs and reading materials including textbook and other assigned readings. Assignments will be graded

Exams: Two exams designed to test the understanding of all material covered in class. Comprehensive review of all course material including textbook, lab exercises, assignments and any additional assigned reading is expected.

Final Project: A group project to help appreciate practicalities involved in the application of theoretical concepts learned in the course. All team member is expected to make fair contribute. Peer review from other members will happen at the end of the project.

Weekly exercises: 10 exercises each week – comprising brief readings, videos, and occasion pop quizzes
